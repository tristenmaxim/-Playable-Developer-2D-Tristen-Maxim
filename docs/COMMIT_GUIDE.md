# Руководство по коммитам для проекта

## Где посмотреть коммиты?

### 1. В терминале (локально)
```bash
# Краткий список коммитов
git log --oneline

# Детальная информация
git log --pretty=format:"%h - %an, %ar : %s" -10

# С изменениями файлов
git log --stat

# Графическое представление
git log --graph --oneline --all
```

### 2. На GitHub
Перейдите на страницу репозитория:
https://github.com/tristenmaxim/-Playable-Developer-2D-Tristen-Maxim

Нажмите на вкладку **"Commits"** или посмотрите историю коммитов в правом верхнем углу.

## Что делает коммит "осмысленным"?

### ✅ Хороший коммит:
1. **Логически завершенная единица работы**
   - Например: "Добавлена базовая структура игры"
   - Или: "Реализована система прыжков персонажа"
   - Или: "Добавлены звуковые эффекты"

2. **Четкое описание изменений**
   - Сообщение коммита должно объяснять ЧТО и ЗАЧЕМ было изменено
   - Используйте формат: `тип: краткое описание`
   - Примеры типов: `feat:`, `fix:`, `refactor:`, `style:`, `docs:`

3. **Не слишком большой и не слишком маленький**
   - ❌ Плохо: "Все изменения" (слишком большой)
   - ❌ Плохо: "Исправлена опечатка" (слишком маленький, если это единственный коммит)
   - ✅ Хорошо: "Добавлена физика прыжка персонажа"

### ❌ Плохой коммит:
- "Обновления" (непонятно что)
- "Фиксы" (непонятно что исправлено)
- "Изменения" (слишком общо)
- Один большой коммит со всеми изменениями

## Текущая ситуация

Сейчас у нас **1 коммит**:
- ✅ Хороший коммит с четким описанием
- ❌ Но недостаточно для требования "несколько коммитов"

## Что нужно сделать?

В процессе разработки игры нужно делать коммиты на каждом этапе:

1. **Начало разработки:**
   - `feat: базовая структура проекта`

2. **Добавление функционала:**
   - `feat: инициализация PixiJS приложения`
   - `feat: загрузка и отображение фона`
   - `feat: добавлен персонаж с базовой анимацией`

3. **Игровая механика:**
   - `feat: реализована система прыжков`
   - `feat: добавлены враги`
   - `feat: система коллизий`

4. **Полировка:**
   - `feat: добавлен счет`
   - `feat: звуковые эффекты`
   - `refactor: оптимизация производительности`

5. **Финализация:**
   - `refactor: оптимизация размера бандла`
   - `docs: финальная документация`

## Рекомендации

- Делайте коммиты после завершения каждой логической части
- Минимум 5-10 коммитов для полноценной разработки
- Каждый коммит должен быть работоспособным (игра должна запускаться)
- Используйте понятные сообщения на русском или английском
